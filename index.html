Design Alignment

A Storage Substrate, Not a Streaming Engine

Nereus is the unified storage layer beneath Pulsar, KoP, and Kafka. It provides pluggable WAL and object-store profiles, generation-based compaction, and lakehouse tiering. Each upper system is an adapter — not a separate storage backend.

Core Design

One Storage Core, Many Upper Systems

Nereus is not a Pulsar variant or a Kafka variant — it is the unified stream storage layer beneath both. Topic-level profiles, generation migration, and protocol-neutral abstractions make it a true substrate.

🔲

Topic-level Storage Profile

Same cluster, different topics, different storage backends. Pick BookKeeper-only, Object-WAL + Object-Store, or Lakehouse — per topic or per namespace.

🔄

Generation Migration

Migrate a topic from BK to Object Store across a generation boundary. No implicit mixing, no dual-write — clean, atomic profile transitions.

📐

Unified streamId + offset

Pulsar MessageId, Kafka offset, Lakehouse row position — all just projections of a single streamId + offset coordinate. No protocol leakage into the storage layer.

⚡

Pluggable WAL Profiles

BookKeeper WAL for low latency, Object WAL for cost efficiency, Local WAL for edge. WAL and final data-store profiles are independently selectable.

🔌

Upper-System Neutral

Pulsar, KoP, and Kafka KRaft are adapters — not Nereus itself. Each adapter translates its protocol into the same StreamStorage API. One storage core, many protocols.

☑️

Metadata-Driven Correctness

A dedicated metadata plane decides visibility, assigns offsets, and fences stale commits. Object stores store bytes — not truth. The broker is stateless for correctness.

🗄️

Stream-Table Duality

SBT exposes streams as built-in Iceberg tables. SDT delivers to external catalogs. Lakehouse queries and streaming reads share the same compacted objects.

📦

Immutable Objects + GC

WAL, compacted, snapshot, and table objects are immutable once committed. GC is driven by metadata-plane reference counting — not by object store list operations.

Implementation Roadmap

7-Phase Plan: From Core to All Adapters

Build the storage core first, then attach adapters. Pulsar native first (lowest risk), KoP next (reuses Pulsar path), Kafka KRaft last (hardest).

0

Project Scaffolding & Module Boundaries Done

Gradle multi-module structure: nereus-api, nereus-core, nereus-storage-bk, nereus-object-store, nereus-managed-ledger, nereus-pulsar-adapter, nereus-kop-adapter. Fork workflow established.

1

Nereus Storage Core Phase 1

StreamStorage API (streamId + offset, append/read/resolve/trim), generation/segment/index, StorageProfile abstraction, metadata plane integration, cursor/checkpoint primitives. No Pulsar or Kafka dependencies at this layer.

2

BookKeeper Profile Phase 2

WalProfile=BOOKKEEPER_WAL + DataProfile=BOOKKEEPER_LEDGER, then BK_WAL + OBJECT_SEGMENT. Fast Pulsar integration, reuse BK's consensus and low-latency properties. Demonstration of WAL+Data profile separation.

3

Object WAL + Object Segment Profiles Phase 3

WalProfile=OBJECT_WAL + DataProfile=OBJECT_SEGMENT. True shared-storage: WAL and final data both on object store. Multi-stream WAL aggregation, per-stream visibility through the metadata offset index.

4

Pulsar ManagedLedger Adapter Phase 4

ManagedLedger + ManagedCursor replacement via topic policy. MessageId → streamId + offset, Position mapping. Topic-level StorageProfile selection. First demonstration of multi-profile coexistence within one cluster.

5

KoP Adapter Phase 5

KoP reuses Pulsar-native storage path — no separate storage. Kafka offset=stream offset, fetch isolation, consumer group mapping, transaction markers, __consumer_offsets handling.

6

Lakehouse Tiering & Compaction Phase 6

DataProfile=OBJECT_SEGMENT+LAKEHOUSE_TABLE. Flink-sync and internal compaction. SBT (built-in) + SDT (external delivery). Iceberg/Paimon table formats, generation overlay, atomic index replacement.

7

Kafka KRaft Adapter Phase 7 · Long-term

Kafka UnifiedLog + LogSegment replacement via nereus-kafka-log-adapter. Topic-partition → Nereus stream, leader epoch projection, KRaft metadata image integration. Hardest path — deferred to later phase.
